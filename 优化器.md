所有优化器的终极目标都是解决最基础的 **SGD（随机梯度下降）** 的痛点：

1. **峡谷震荡：** 在不同方向坡度差异大的地方，来回震荡，下山极慢。
2. **陷入困境：** 容易卡在局部极小值（Local Minima）或鞍点（Saddle Point）。
3. **众生平等：** 所有参数共享同一个学习率，无法做到“因材施教”。

### 第一家族：动量派（致力于解决“方向与震荡”问题）

#### 1. Momentum (标准动量)

**核心思想：** 模拟物理学中的滚石下山。不仅看当前的坡度（梯度），还要继承之前的速度（惯性）。
**解决了什么：** 解决了 SGD 在峡谷地形中的“之”字形震荡问题，并能借助惯性冲出较浅的局部极小值。

**数学原理：**


*  是速度（Velocity），初始为 0。
*  (rho) 是摩擦系数（通常设为 **0.9**），防止速度无限累加。
*  是学习率， 是当前梯度。

**代码实现：**

```python
# 初始化：v = np.zeros_like(x)
v = rho * v - learning_rate * dx
x += v

```

**优点：** 收敛速度显著快于基础 SGD，有效抑制震荡。
**不足：** “盲目自信”。在快冲到谷底时，因为惯性太大容易刹不住车，冲过头（Overshooting）。

---

#### 2. Nesterov Momentum (NAG)

**核心思想：** 既然有惯性，那我不如先“预判”一下依靠惯性会冲到哪里，然后去那个“未来位置”计算梯度，提前修正现在的方向。
**解决了什么：** 解决了标准 Momentum 容易“冲过头”的问题，刹车更灵敏。

**数学原理（换元法优化后）：**
为了在代码中不重复计算前向传播，我们进行变量代换，直接基于当前位置的梯度  进行更新：


**代码实现：**

```python
# 初始化：v = np.zeros_like(x)
v_prev = v  # 保存上一步的速度
v = rho * v - learning_rate * dx
x += -rho * v_prev + (1 + rho) * v

```

**优点：** 具有理论上更强的收敛保证（对于凸函数），实际应用中比普通 Momentum 更快、震荡更小。
**不足：** 依然无法解决“所有参数共享同一个学习率”的问题。

### 第二家族：自适应学习率派（致力于解决“步长与维度差异”问题）

#### 3. AdaGrad (Adaptive Gradient)

**核心思想：** 给每个参数定制学习率。给每个参数准备一个“记仇本”，记录它历史上所有梯度的平方和。历史波动越大的参数，我就强行把它的学习率压低；历史波动小的参数，我就给它大一点的学习率。
**解决了什么：** 解决了稀疏特征更新缓慢的问题，无需手动精细调优学习率。

**数学原理：**


**代码实现：**

```python
# 初始化：cache = np.zeros_like(x)
cache += dx ** 2
x += -learning_rate * dx / (np.sqrt(cache) + eps)

```

**优点：** 对稀疏数据极其友好，能自动调整每个参数的步伐。
**不足（致命缺陷）：** 因为 `cache` 是一直单调累加的，到了训练中后期，分母会变得极其巨大，导致有效学习率趋近于 **0**，模型彻底停止学习（过早收敛）。

---

#### 4. RMSProp (Root Mean Square Propagation)

**核心思想：** 拯救 AdaGrad。把“死记硬背的死账本”改成“会遗忘的滑动平均账本”。只关注最近一段时间的梯度波动，太久远的历史就让它随风而去。
**解决了什么：** 完美解决了 AdaGrad 学习率单调递减到 0 的致命问题。

**数学原理：**
引入衰减率  (decay_rate，通常设为 **0.99**)。


**代码实现：**

```python
# 初始化：cache = np.zeros_like(x)
cache = decay_rate * cache + (1 - decay_rate) * dx ** 2
x += -learning_rate * dx / (np.sqrt(cache) + eps)

```

**优点：** 在非凸优化（如深度神经网络）中表现极佳，学习率能动态自适应，不会提前停止。
**不足：** 它只调整了“步伐大小（幅度）”，但丢掉了 Momentum 家族的“惯性（方向）”优势。

### 第三家族：终极缝合怪（集大成者）

#### 5. Adam (Adaptive Moment Estimation)

**核心思想：** 小孩子才做选择，我全都要！把 Momentum 的“惯性（一阶动量）”和 RMSProp 的“自适应步长（二阶动量）”结合起来，并加上开局的“防沉迷”保护（偏差校正）。
**解决了什么：** 综合了所有算法的优点，几乎做到了“开箱即用”，对超参数极其不敏感。

**数学原理：**

1. **一阶动量 (方向)：** 
2. **二阶动量 (幅度)：** 
3. **偏差校正 (去冷启动)：** ， 
4. **参数更新：** 

**代码实现：**

```python
# 初始化：m = np.zeros_like(x), v = np.zeros_like(x), t = 0
t += 1
m = beta1 * m + (1 - beta1) * dx
v = beta2 * v + (1 - beta2) * (dx ** 2)

# 偏差校正
m_unbias = m / (1 - beta1 ** t)
v_unbias = v / (1 - beta2 ** t)

# 更新
x += -learning_rate * m_unbias / (np.sqrt(v_unbias) + eps)

```

**优点：** * 极快的收敛速度。

* 结合了方向指引与自适应步长。
* 偏差校正机制让模型在训练初期极其稳定。
* 目前 90% 以上深度学习任务的首选默认优化器（通常设置 ）。

**不足之处：**

* 计算代价和内存占用稍大（需要为每个参数维护  和  两个状态变量）。
* 在某些特定的计算机视觉任务（如 ResNet 图像分类）上，经过精心调参的 SGD+Momentum 最终泛化性能可能会略高于 Adam（因为 Adam 可能会收敛到较尖锐的局部极小值）。为了解决这个问题，后来又进化出了 **AdamW**（加入了权重衰减的解耦）。

### 总结大比拼

| 优化器 | 解决的主要痛点 | 核心机制 | 适用场景 |
| --- | --- | --- | --- |
| **Momentum** | 峡谷震荡，局部极小值 | 累加历史梯度 (惯性) | 需要稳定方向、常规网络 |
| **Nesterov** | Momentum 容易冲过头 | 提前在未来位置算梯度 | 替代普通 Momentum |
| **AdaGrad** | 需要手动调学习率，稀疏特征 | 累加历史梯度平方作为惩罚 | NLP、稀疏特征任务 |
| **RMSProp** | AdaGrad 学习率掉到 0 | 滑动平均计算梯度平方 | RNN、通用深度网络 |
| **Adam** | 以上所有痛点 | 一阶动量 + 二阶动量 + 校正 | **无脑首选**，几乎所有模型 |
